# 企业级 AI Agent 竞品销售话术清单

> 每个卡片按“定位 / 优势 / 限制 / 对比帝王蟹切入点”组织，供销售现场复制。

## 🦀 网易智企 帝王蟹 ClawHive
- 一句话定位 — 企业级 AI Agent 的“管控中枢”，把分散龙虾和自研 Agent 纳入统一权限、安全审计与技能沉淀。
- 三个核心优势
  - 生态中立 — 客户用什么 IM 就接什么 IM，不要求重构现有办公入口。
  - 模型中立 — 可按业务切换主流模型，避免被单一模型路线锁死。
  - 安全治理 — 以权限、数据隔离、审计、合规为核心，让企业敢让 AI 碰核心数据。
- 三个主要限制
  - 认知门槛 — 客户先要理解“管控层”价值，不是单纯买一个办公助手。
  - 初期配置 — 权限、技能、审计策略需要 IT 与业务共同梳理。
  - 产品窗口早期 — 新发布产品，行业案例与标准报价需要继续向厂商确认。
- 跟帝王蟹 ClawHive 对比切入点 — 核心话术：别人解决“让 AI 干活”，帝王蟹解决“让企业敢放开 AI 干活”。
- 适合触发场景 — 多 IM 并存、员工私用 AI、强监管合规、需要集团级统一治理。

## 🐧 腾讯 WorkBuddy / QClaw
- 一句话定位 — 腾讯系“免部署办公龙虾”，让员工通过 IM 远程指挥电脑处理报表、文档、会议等办公任务。
- 三个核心优势
  - 开箱即用 — 主打免部署、一键安装，销售沟通时容易讲清楚。
  - 腾讯协同 — 企业微信、腾讯文档、腾讯会议等生态内流程更顺。
  - 多入口可用 — 支持多 IM 遥控和多模型切换，个人办公提效感强。
- 三个主要限制
  - 偏执行助手 — 解决“替员工干活”，不是集团级 Agent 治理中枢。
  - 生态偏向 — 腾讯生态内体验更强，跨生态治理价值有限。
  - 强监管需确认 — 细粒度权限、全链路审计、本地专属能力要逐项核验。
- 跟帝王蟹 ClawHive 对比切入点 — 对比帝王蟹时不要说它弱；说 WorkBuddy 是执行层，ClawHive 是管控层。客户问“全集团谁能管住所有 AI”，切回帝王蟹。
- 适合触发场景 — 客户已经重度使用企业微信、腾讯文档，目标是先做办公提效。

## ☁️ 字节火山引擎 ArkClaw
- 一句话定位 — 云端 SaaS 版 OpenClaw，给中小企业和飞书用户低门槛、网页即用地“云上养龙虾”。
- 三个核心优势
  - 零部署 — 打开网页即可用，适合快速试错和轻量团队。
  - 云端在线 — 7×24 小时云端资源，不受员工本地电脑在线状态影响。
  - 飞书适配 — 与飞书文档、会议、任务等办公入口结合更自然。
- 三个主要限制
  - 纯云端架构 — 金融、医疗、能源等数据不出域场景会有硬门槛。
  - 内网系统受限 — 本地老旧系统、内网客户端、复杂权限域穿透能力需验证。
  - 治理深度有限 — 更像托管智能体平台，不是跨厂商 Agent 管控总线。
- 跟帝王蟹 ClawHive 对比切入点 — 客户要“快试、低成本、飞书内跑起来”，ArkClaw 好讲；客户要“核心数据不出域、统一审计、集团治理”，帝王蟹更好切。
- 适合触发场景 — 客户是飞书用户、预算敏感、先做 PoC，不涉及核心敏感数据。

## 🦞 智谱 AutoClaw（澳龙）
- 一句话定位 — 本地一键安装版 OpenClaw，把原本复杂的开源部署变成普通员工也能安装的桌面智能体。
- 三个核心优势
  - 部署极简 — 下载安装、登录即可用，降低 OpenClaw 上手门槛。
  - 本地执行 — 任务在本机运行，适合对云端上传敏感的个人或小团队。
  - 技能预置 — 预置常用 Skills，并接入智谱模型与浏览器自动化能力。
- 三个主要限制
  - 组织管控弱 — 本地安全不等于企业级权限、审计、审批和追责。
  - 规模化难管 — 员工各自安装后，版本、Skill、数据访问边界容易分散。
  - 依赖自管 — 企业要自行处理技能安全、日志留存和合规证明。
- 跟帝王蟹 ClawHive 对比切入点 — AutoClaw 可作为“执行层龙虾”；帝王蟹负责“管一整支龙虾军团”。两者可互补，不必硬碰硬。
- 适合触发场景 — 客户强调本地安装、个人效率、技术团队先试用。

## 📌 阿里钉钉 AI 助理
- 一句话定位 — 钉钉生态内的 AI 办公入口，适合已把审批、沟通、客服和组织协同放在钉钉上的企业。
- 三个核心优势
  - 零切换 — 钉钉用户无需换入口，员工接受成本低。
  - 云钉一体 — 与阿里云算力和钉钉流程结合，适合阿里系客户。
  - 办公流程深 — 审批、客服、通知、文档等钉钉内场景推进快。
- 三个主要限制
  - 生态封闭 — 跨飞书、企微、自建 IM 的统一治理不是核心强项。
  - 模型路线固定 — 客户容易被通义和阿里云体系绑定。
  - 外部系统穿透 — 非阿里系垂直软件、本地客户端和复杂老系统需验证。
- 跟帝王蟹 ClawHive 对比切入点 — 客户用钉钉不等于只能买钉钉 AI。帝王蟹可以保留钉钉入口，同时管理飞书、企微、自建 IM 和第三方 Agent。
- 适合触发场景 — 客户全员钉钉，想先在审批、客服、协同里上 AI。

## 🪽 字节飞书 aily / 智能伙伴
- 一句话定位 — 飞书文档、多维表格和业务系统里的智能伙伴，适合以飞书作为组织协作中枢的企业。
- 三个核心优势
  - 文档协作强 — 与飞书文档、多维表格、知识库的结合天然顺。
  - 业务对接快 — 支持通过 MCP 等方式连接业务系统，适合流程自动化。
  - 可定制形象 — 企业可配置 Agent 工作台和角色，便于内部推广。
- 三个主要限制
  - 迁移成本 — 非飞书客户为了用 AI 换 IM，决策阻力大。
  - 生态边界 — 更擅长飞书域内协同，跨 IM 管控不是主战场。
  - 治理覆盖有限 — 对第三方龙虾、自研 Agent 的统一权限与审计需另配。
- 跟帝王蟹 ClawHive 对比切入点 — 飞书 aily 是“飞书里的智能伙伴”；帝王蟹是“所有智能伙伴的管理者”。集团多入口、多 Agent 时，帝王蟹更好切入。
- 适合触发场景 — 客户飞书深度使用，AI 先从文档、表格、知识库落地。

## 🧱 百度千帆 AppBuilder
- 一句话定位 — 企业级 AI 应用开发管理平台，帮助企业构建 RAG、Agent、工作流和智能应用。
- 三个核心优势
  - 工具链完整 — RAG、Agent、工作流、UI Builder 等能力适合开发团队。
  - 低代码到全代码 — 覆盖不同开发能力，从试点到深度定制都能承接。
  - 政企经验 — 在政务、金融、电力、能源等行业有较多应用场景。
- 三个主要限制
  - 偏“做 Agent” — 核心是开发平台，不是专门管控所有 Agent 的治理层。
  - 需要建设能力 — 知识库、流程、应用上线仍需要技术团队参与。
  - 前端入口分散 — IM 远程执行、员工端使用管控需结合具体系统实现。
- 跟帝王蟹 ClawHive 对比切入点 — AppBuilder 帮客户“做出 Agent”，帝王蟹帮客户“管住 Agent”。技术团队强时可共存：一个建设，一个治理。
- 适合触发场景 — 客户重点是开发知识库问答、业务应用、RAG 或内部工具。

## 🤖 实在智能 · 实在 Agent
- 一句话定位 — Agentic RPA 路线，擅长让 AI 操作没有 API 的老旧系统、网页后台和重复流程。
- 三个核心优势
  - 执行能力强 — 视觉理解 + RPA 适合 ERP、政务网、电商后台等复杂界面。
  - 场景落地深 — 在消费品、电商、运营流程等行业有实际客户案例。
  - 替代人工明显 — 适合高频、规则相对稳定、人工重复操作多的岗位。
- 三个主要限制
  - 偏执行层 — 重点是自动化完成任务，不是统一治理所有企业 AI。
  - 流程梳理重 — 落地前要拆流程、定规则、处理异常分支。
  - 通用管控需集成 — 跨 IM、跨模型、跨 Agent 的统一权限审计要另行设计。
- 跟帝王蟹 ClawHive 对比切入点 — 实在智能适合当“会干脏活累活的执行层”；帝王蟹适合当“统一调度和审计层”。客户老系统多时可组合销售。
- 适合触发场景 — 客户有大量人工录入、后台操作、无 API 系统自动化需求。

## 🏦 蚂蚁数科 Agentar
- 一句话定位 — 面向金融行业的企业级 Agent 方案，强调蚂蚁在金融数据、风控和合规场景上的沉淀。
- 三个核心优势
  - 金融理解 — 对支付、风控、合规、运营等金融流程理解更深。
  - 行业背书 — 蚂蚁在金融科技场景的可信度和客户资源强。
  - 场景专精 — 适合围绕金融业务做垂直智能体落地。
- 三个主要限制
  - 泛行业弱化 — 离开金融主场后，通用企业场景优势会下降。
  - 生态依赖 — 客户可能进入蚂蚁金融科技体系的集成路径。
  - 管控边界 — 多 IM、多厂商 Agent 的集团级统一治理不是核心卖点。
- 跟帝王蟹 ClawHive 对比切入点 — 金融业务智能体可看 Agentar；企业级“所有 AI 统一可管可审计”仍可切帝王蟹，尤其是集团多系统并存。
- 适合触发场景 — 客户是金融机构，关注风控、合规、支付、运营智能化。

## 🔎 明略科技 DeepMiner
- 一句话定位 — 面向集团级企业的数据洞察和复杂决策平台，强调可信智能、透明可追溯和幻觉控制。
- 三个核心优势
  - 数据洞察强 — 适合复杂业务分析、集团经营看板和决策辅助。
  - 可信可追溯 — 强调全流程透明、降低模型幻觉对决策的影响。
  - 企业级项目经验 — 适合数据量大、组织层级复杂的集团客户。
- 三个主要限制
  - 偏分析决策 — 不是以 IM 遥控、本地执行和员工端 Agent 为核心。
  - 数据治理前置 — 效果依赖客户现有数据质量、口径和系统连接。
  - 管控对象不同 — 重点是数据智能，不是统一管理 OpenClaw/第三方 Agent。
- 跟帝王蟹 ClawHive 对比切入点 — DeepMiner 解决“看清业务”，帝王蟹解决“管住 AI 操作”。客户同时关心数据分析与 AI 治理时，可分层讲。
- 适合触发场景 — 客户关注经营洞察、复杂分析、集团决策透明化。

## 🧩 BetterYeah AI
- 一句话定位 — 企业级 AI 智能体开发与编排平台，偏工作流、知识库、多 Agent 协同和 LLMOps。
- 三个核心优势
  - 编排能力强 — NeuroFlow 等工作流能力适合复杂业务流程。
  - 知识库能力 — VisionRAG、多模态知识库适合内容密集型企业。
  - 模型覆盖广 — 支持多模型接入和 LLMOps，技术团队可控性高。
- 三个主要限制
  - 实施要求高 — 要发挥价值，需要流程设计、知识治理和平台运维能力。
  - 偏建设平台 — 更像“搭应用和编排流程”，不是专门做企业 AI 监管。
  - 合规需核验 — 强监管场景的本地部署、审计、权限深度要逐项确认。
- 跟帝王蟹 ClawHive 对比切入点 — BetterYeah 适合“建复杂 Agent 应用”；帝王蟹适合“管住企业里所有 Agent”。技术团队强的客户可谈互补。
- 适合触发场景 — 客户要自建复杂 AI 应用、多 Agent 流程和知识库。
